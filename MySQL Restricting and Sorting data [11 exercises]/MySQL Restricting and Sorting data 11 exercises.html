  • 1. Write a query to display the name (first_name, last_name) and salary for all employees whose salary is not in the range $10,000 through $15,000.
  • Idea, Solution, Result
    ## Idea

- 이름 쿼리 할 것
- salary가 10000 미만, 15000 초과인 사람 구할 것
  - between 써도 될 듯, between은 inclusive한 개념임, 단 날짜 쓸 때는 조심 할 것
  - [between 참고](http://kwonnam.pe.kr/wiki/database/mysql/between)
  - between 앞에 not을 쓰면 범위 밖임
    ## solution

select concat(FIRST_NAME, ' ', LAST_NAME) as employee_name,
       SALARY
from employees
where SALARY > 15000 or SALARY < 10000;

select concat(FIRST_NAME, ' ', LAST_NAME) as employee_name,
       SALARY
from employees
where SALARY NOT between 10000 and 15000;

  • 2. Write a query to display the name (first_name, last_name) and department ID of all employees in departments 30 or 100 in ascending order.
  • Idea, Solution, Result
    ## Idea

- 이름은 쿼리 하면 됨

- where 조건에 department_id in (30, 100) 하면 포함 되는 구조

       LAST_NAME,

  • 5. Write a query to display the first_name of all employees who have both "b" and "c" in their first name.
  • Idea, Solution, Result
    ## Idea

- first_name에 'b', 'c'를 가진 사람을 찾으면 됩니다.
- like로 패턴 매칭을 하면 될 듯
  - 양방향 매칭을 해야함
- regexp로 패턴 매칭도 가능 할 듯
  - regexp에서 여러개를 찾으려면 and를 쓰면 된다.
  - [regexp and](https://stackoverflow.com/questions/15687136/logical-and-operator-in-mysql-regexp)
    ## solution

select FIRST_NAME
from employees
where FIRST_NAME like '%b%' 
  and FIRST_NAME like '%c%';

select FIRST_NAME
from employees
where FIRST_NAME regexp '[b]'
  and FIRST_NAME regexp '[c]';

  • 10. Write a query to display the name (first_name, last_name), salary and PF (15% of salary) of all employees.
  • Idea, Solution, Result
    ## Idea

- 했던 문제 같은데...
- first_name, last_name, salary, salary * 0.15 구하면 될 듯
  - salary는 round를 써보았다.
    ## solution

select FIRST_NAME,
       LAST_NAME,
       round(SALARY, 0) as Salary,
       cast(SALARY * 0.15 as decimal(10, 0)) as PF
from employees
